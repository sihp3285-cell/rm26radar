# 卡尔曼滤波器架构深度剖析

这份文档从**数学原理到工程实现**，深度剖析本项目中卡尔曼滤波器的架构设计。不仅讲解"怎么用"，更讲解"为什么这样设计"以及"工程中踩过的坑"。

# 一、卡尔曼滤波器在本项目中的角色

```text
┌─────────────────────────────────────────────────────┐
│                    Tracker 子系统                     │
│                                                      │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────┐ │
│  │ KalmanFilterBox│  │ KalmanFilter2d│  │ BotIdentity │ │
│  │  8维像素框平滑  │  │  4维世界坐标平滑│  │ 跨帧身份稳定 │ │
│  └──────┬───────┘  └──────┬───────┘  └──────┬─────┘ │
│         │                  │                  │       │
│         └──────────┬───────┘                  │       │
│                    │                          │       │
│              ┌─────┴─────┐                    │       │
│              │   Tracker  │←───────────────────┘       │
│              │ 固定槽位管理 │                            │
│              └───────────┘                            │
│                    ↑                                  │
│           WorldMeasurement                            │
│          (检测框 + 世界坐标)                            │
└─────────────────────────────────────────────────────┘
```

本项目使用了**两个独立的卡尔曼滤波器**，分别在不同坐标空间做平滑：

| 滤波器 | 状态维度 | 观测维度 | 坐标空间 | 用途 |
|--------|---------|---------|---------|------|
| `KalmanFilterBox` | 8 维 | 4 维 | 像素坐标 | 检测框位置和大小平滑 |
| `KalmanFilter2d` | 4 维 | 2 维 | 世界坐标 | 场地坐标平滑 |

两者都采用**匀速运动模型（Constant Velocity Model）**，配合 BotIdentity 的**语义空间平滑**，构成完整的三维平滑体系。

# 二、为什么用两个独立滤波器？

## 2.1 错误方案：一个 10 维滤波器

有人可能会想：为什么不把像素框和世界坐标塞进一个滤波器？

```text
状态 = [cx, cy, w, h, world_x, world_z, vx, vy, vw, vh]  // 10 维
```

这个方案有三个致命问题：

### 问题 1：坐标系非线性

像素坐标和世界坐标之间的关系是**非线性的**（涉及相机内参、外参、畸变、PnP 解算）。卡尔曼滤波器要求状态转移是线性的（`x = F·x`），非线性关系需要扩展卡尔曼（EKF）或无迹卡尔曼（UKF），复杂度大幅上升。

### 问题 2：噪声特性不同

| 空间 | 噪声来源 | 典型量级 |
|------|---------|---------|
| 像素坐标 | 检测模型的框抖动 | ±3~10 像素 |
| 世界坐标 | PnP 解算误差 | ±0.1~0.5 米 |

两种噪声的量级差 10~100 倍。如果用一个滤波器，`R` 矩阵对角线上的元素会相差百倍，导致数值不稳定。

### 问题 3：更新频率可能不同

某些帧可能有像素检测但 PnP 解算失败（如标定不准）。分开后可以独立控制更新频率。

## 2.2 正确方案：两个独立滤波器

```text
KalmanFilterBox：像素空间平滑（检测框 → 平滑框）
KalmanFilter2d：世界空间平滑（原始世界坐标 → 平滑世界坐标）
```

两个滤波器**完全独立**，各自有自己的 `F`、`H`、`Q`、`R`、`P` 矩阵，互不影响。

# 三、匀速运动模型（Constant Velocity Model）

## 3.1 状态向量

### KalmanFilterBox（8 维）

```text
x = [cx, cy, w, h, vx, vy, vw, vh]ᵀ
     ├── 位置 4维 ──┤  ├── 速度 4维 ──┤
```

* `cx, cy`：检测框中心的像素坐标
* `w, h`：检测框的宽和高（像素）
* `vx, vy`：中心点的像素速度（像素/帧）
* `vw, vh`：宽高的变化速度（像素/帧）

### KalmanFilter2d（4 维）

```text
x = [x, z, vx, vz]ᵀ
     ├── 位置 2维 ─┤  ├── 速度 2维 ─┤
```

* `x, z`：世界坐标（米）
* `vx, vz`：世界坐标系下的速度（米/帧）

---

## 3.2 状态转移矩阵 F

匀速运动模型的核心假设：

> **目标在两帧之间做匀速直线运动。**

```text
x_new = x_old + v × dt
v_new = v_old（速度不变）
```

对于 KalmanFilterBox：

```text
F = [1  0  0  0  dt 0  0  0 ]    cx_new  = cx + vx × dt
    [0  1  0  0  0  dt 0  0 ]    cy_new  = cy + vy × dt
    [0  0  1  0  0  0  dt 0 ]    w_new   = w  + vw × dt
    [0  0  0  1  0  0  0  dt]    h_new   = h  + vh × dt
    [0  0  0  0  1  0  0  0 ]    vx_new  = vx
    [0  0  0  0  0  1  0  0 ]    vy_new  = vy
    [0  0  0  0  0  0  1  0 ]    vw_new  = vw
    [0  0  0  0  0  0  0  1 ]    vh_new  = vh
```

KalmanFilter2d 的 `F` 结构完全对称，只是从 8×8 降到了 4×4。

---

## 3.3 为什么匀速模型适合 RoboMaster？

在 RoboMaster 比赛中：

* 车速约 3~5 m/s
* 帧率 30~60 fps
* 相邻帧间位移约 5~17 cm

对于如此短的时间间隔（16~33ms），匀速假设非常合理。车辆的加速度在单帧内造成的位移偏差通常 < 1cm，远小于检测噪声。

---

# 四、观测模型

## 4.1 观测矩阵 H

```text
对于 KalmanFilterBox（8维状态，4维观测）：

H = [1  0  0  0  0  0  0  0 ]    观测到 cx
    [0  1  0  0  0  0  0  0 ]    观测到 cy
    [0  0  1  0  0  0  0  0 ]    观测到 w
    [0  0  0  1  0  0  0  0 ]    观测到 h
```

我们只能**直接观测**位置 `[cx, cy, w, h]`，无法直接观测速度 `[vx, vy, vw, vh]`。

速度信息是通过**多帧位置变化**隐式推断出来的——这正是卡尔曼滤波器的核心价值。

---

## 4.2 测量噪声 R

```cpp
R = Eigen::Matrix<float, 4, 4>::Identity() * (r_std * r_std);
```

默认 `r_std = 1.0`，所以 `R = I`（对角矩阵，对角线为 1.0）。

这意味着假设检测框的噪声标准差为 1 像素。在实际中，YOLO 检测框的抖动通常在 3~10 像素，可以通过构造函数参数调整 `r_std`。

---

# 五、过程噪声 Q 的动态计算

## 5.1 连续时间白噪声模型

```cpp
void KalmanFilterBox::updateQ(float dt) {
    float dt2 = dt * dt;
    float dt3 = dt2 * dt / 2.0f;
    float dt4 = dt2 * dt2 / 4.0f;

    Q.setZero();
    for (int i = 0; i < 4; i++) {
        Q(i, i) = dt4;           // 位置-位置
        Q(i, i+4) = dt3;         // 位置-速度
        Q(i+4, i) = dt3;         // 速度-位置
        Q(i+4, i+4) = dt2;       // 速度-速度
    }
    Q *= (q_std_ * q_std_);
}
```

对于每个维度，Q 的 2×2 子块为：

```text
Q_i = q² × [dt⁴/4   dt³/2]
            [dt³/2   dt²   ]
```

这是从连续时间随机加速度模型离散化得到的。数学推导基于：

> 假设加速度是一个零均值白噪声过程，其功率谱密度为 q²。

## 5.2 为什么要在 predict 里重新计算 Q？

```cpp
std::vector<float> KalmanFilterBox::predict(float dt) {
    float use_dt = (dt > 0) ? dt : dt_;
    for(int i=0; i<4; i++) F(i, i+4) = use_dt;
    updateQ(use_dt);
    // ...
}
```

因为 `dt` 可能每帧不同（帧率波动、丢帧）。`Q` 必须随 `dt` 动态调整。

如果 `dt` 翻倍：

* `Q` 的位置分量增大 16 倍（`dt⁴`）
* 速度分量增大 4 倍（`dt²`）

这意味着长时间没观测时，不确定性增长更快，卡尔曼增益会更大，滤波器更信任新的观测——这正是我们期望的行为。

# 六、卡尔曼增益的物理意义

## 6.1 标准更新公式

```text
S = H·P·Hᵀ + R         // 新息协方差
K = P·Hᵀ·S⁻¹            // 卡尔曼增益
x = x + K·(z - H·x)     // 状态更新
P = (I - K·H)·P          // 协方差更新
```

## 6.2 增益 K 的直觉理解

卡尔曼增益 `K` 决定了**"更信任观测还是更信任预测"**：

```text
K → 0：完全信任预测（模型很准，观测噪声大）
K → 1：完全信任观测（模型不准，观测噪声小）
```

影响 `K` 的两个因素：

| 因素 | 变化 | K 的变化 | 含义 |
|------|------|---------|------|
| `R`（测量噪声）增大 | 观测更不可信 | K 减小 | 更信任预测 |
| `P`（预测不确定性）增大 | 预测更不可信 | K 增大 | 更信任观测 |

---

## 6.3 新息（Innovation）

```text
新息 = z - H·x
```

即**观测值与预测值的差异**。新息的大小反映了模型的预测质量：

* 新息小 → 预测准确，滤波器状态良好
* 新息大 → 预测偏差大，可能需要更多信任观测

---

# 七、工程级防跳变机制

## 7.1 问题：异常观测导致滤波器发散

教科书上的卡尔曼滤波器假设观测噪声是高斯分布的。但在实际工程中，检测模型的输出经常有**异常值**：

* ID 切换：上一帧检测到 Red R1，这一帧检测到 Blue R3（同一位置）
* 误检：突然出现一个远离目标的检测框
* 漏检后重匹配：目标被遮挡后重新出现，位置可能跳变

如果不做保护，一个异常观测就能让滤波器状态被污染，后续所有帧都跟着偏。

## 7.2 解决：KalmanFilterBox 的防跳变

```cpp
std::vector<float> KalmanFilterBox::update(const std::vector<float>& bbox) {
    // 防跳变：中心偏差超过 100 像素 → 直接重置
    if (std::abs(x(0) - bbox[0]) > 100.0f || std::abs(x(1) - bbox[1]) > 100.0f) {
        reset(bbox);
        return bbox;
    }
    // ... 正常卡尔曼更新 ...
}
```

阈值 100 像素的工程含义：在 5K 分辨率下，一个目标在相邻帧之间不可能跳动 100 像素以上。

## 7.3 解决：KalmanFilter2d 的防跳变

```cpp
float jump_dist = std::hypot(x(0) - pos[0], x(1) - pos[1]);
if (jump_dist > 5.0f) {
    reset(pos);
    return pos;
}
```

阈值 5 米的工程含义：在 RoboMaster 场地（28m × 15m）上，一辆车在相邻帧之间不可能移动 5 米以上。

## 7.4 为什么 reset 而不是 ignore？

| 策略 | 行为 | 优缺点 |
|------|------|--------|
| `ignore`（丢弃异常观测） | 继续用旧状态预测 | 旧状态可能已经偏了，累积误差 |
| `reset`（以异常观测为准） | 从新位置重新开始 | 信任了一次"可疑"观测，但不会累积误差 |

`reset` 是更安全的选择：即使异常观测是错的，下一帧的正常观测会把滤波器拉回来。而 `ignore` 如果旧状态已经偏了，会一直偏下去。

# 八、初始协方差 P 的设计

```cpp
P = Eigen::Matrix<float, 8, 8>::Identity() * 100.0f;
```

初始不确定性设得很大（100），表示"刚开始时我们对状态一无所知"。

效果：前几帧的卡尔曼增益 `K` 会很大（更信任观测），滤波器快速收敛到实际位置。

如果 `P` 设得太小（如 1），滤波器一开始会过度信任预测（全零），导致收敛缓慢。

---

# 九、`float` vs `double` 的选择

```cpp
Eigen::Matrix<float, 8, 8> F;  // float
```

对于实时视觉跟踪，`float` 的精度完全足够：

* 占用内存更少（矩阵小一半）
* SIMD 运算更快（SSE/AVX 指令一次处理 4/8 个 float）
* GPU 管线通常也用 `float`
* 像素坐标和世界坐标的精度需求不需要 `double`

---

# 十、固定大小矩阵的性能优势

```cpp
Eigen::Matrix<float, 8, 8> F;  // 栈上分配
// vs
Eigen::MatrixXf F(8, 8);       // 堆上分配
```

| 特性 | 固定大小 | 动态大小 |
|------|---------|---------|
| 内存分配 | 栈上（编译期确定） | 堆上（运行时 malloc） |
| 分配速度 | 零开销 | ~100ns/次 |
| 缓存友好 | 连续栈内存 | 可能碎片化 |
| 循环展开 | 编译器可以展开 | 不能展开 |

在每帧跟踪中，`predict` 和 `update` 各被调用 NUM_SLOTS（10）次。固定大小矩阵避免了 20 次堆分配。

# 十一、Tracker 中的状态机与 Kalman 的配合

## 11.1 四状态机

```text
ACTIVE → PREDICTED → LOST → DEAD
```

| 状态 | Kalman 行为 | 是否更新 | 是否外推 |
|------|------------|---------|---------|
| ACTIVE | predict + update | ✅ 观测更新 | ✅ |
| PREDICTED | predict only | ❌ 无观测 | ✅ 外推 |
| LOST | predict only | ❌ 无观测 | ✅ 外推 |
| DEAD | 无操作 | ❌ | ❌ |

## 11.2 ACTIVE 状态的双坐标输出

```cpp
// get_slot 中
out.smoothed_world = (s.state == TrackState::ACTIVE) 
    ? s.detected_world   // ACTIVE：用原始检测坐标
    : s.last_world;      // PREDICTED/LOST：用 Kalman 外推
```

为什么 ACTIVE 时用原始坐标而非 Kalman 平滑值？

在 ACTIVE 状态下，每帧都有可靠的检测结果。此时 Kalman 平滑值和原始值差异很小（因为 `update` 会快速修正）。但 Kalman 值可能有 1~2 帧的融合延迟，对于地图显示来说，使用原始坐标可以让目标点更"跟手"。

当进入 PREDICTED/LOST 状态时（没有新检测），才使用 Kalman 外推值填补空缺。

# 十二、从卡尔曼架构学到的设计要点

## 1. 分而治之

8 维 + 4 维的分离设计，比一个 10 维滤波器更稳定、更灵活、更容易调试。

## 2. 模型要简单

匀速运动模型是最简单的运动模型，但对于 RoboMaster 的帧率和车速来说足够了。更复杂的模型（如加速度模型）会引入更多参数，调参更困难。

## 3. 工程保护比数学公式更重要

防跳变、初始协方差、Q 的动态计算——这些工程技巧的实际价值远大于教科书上的标准公式。

## 4. 状态机控制滤波器生命周期

不是所有帧都需要卡尔曼滤波。通过状态机控制滤波器的行为（更新/外推/冻结/释放），避免了无谓的计算和错误的外推。

## 5. 坐标空间选择影响平滑效果

像素空间平滑消除了框抖动，世界空间平滑消除了坐标抖动，语义空间平滑消除了分类跳变。三者各司其职，互不干扰。
