# 02 ROS2 C++ 节点开发

> 本章深入讲解如何基于 `rclcpp` 编写完整的 ROS2 C++ 节点，涵盖发布者、订阅者、服务、定时器、参数、异常处理等本项目中的所有实战模式。

## 2.1 节点的基本结构

### 2.1.1 最小节点示例

```cpp
#include <rclcpp/rclcpp.hpp>

class MinimalNode : public rclcpp::Node {
public:
    MinimalNode() : Node("minimal_node") {
        RCLCPP_INFO(this->get_logger(), "节点已启动");
    }
};

int main(int argc, char** argv) {
    rclcpp::init(argc, argv);                    // 初始化 ROS2
    auto node = std::make_shared<MinimalNode>(); // 创建节点实例
    rclcpp::spin(node);                          // 进入事件循环
    rclcpp::shutdown();                          // 关闭 ROS2
    return 0;
}
```

### 2.1.2 本项目的节点模式

本项目所有节点都遵循统一模式：

```cpp
class XXXNode : public rclcpp::Node {
public:
    explicit XXXNode(const rclcpp::NodeOptions& options = rclcpp::NodeOptions())
        : Node("xxx_node", options)
    {
        // 1. 声明并读取参数
        // 2. 初始化业务对象（TensorRT 模型、配置管理器等）
        // 3. 创建发布者/订阅者/服务/定时器
        // 4. 打印初始化日志
    }

private:
    // 回调函数
    void topic_callback(const MsgType::ConstSharedPtr msg);
    
    // 成员变量（智能指针管理生命周期）
    rclcpp::Publisher<MsgType>::SharedPtr pub_;
    rclcpp::Subscription<MsgType>::SharedPtr sub_;
    rclcpp::Service<SrvType>::SharedPtr service_;
    rclcpp::TimerBase::SharedPtr timer_;
};

// Component 注册（用于 ComposableNodeContainer）
#include <rclcpp_components/register_node_macro.hpp>
RCLCPP_COMPONENTS_REGISTER_NODE(XXXNode)

// 独立运行入口（向后兼容）
int main(int argc, char** argv) {
    rclcpp::init(argc, argv);
    auto node = std::make_shared<XXXNode>();
    rclcpp::spin(node);
    rclcpp::shutdown();
    return 0;
}
```

## 2.4 服务（Service）

### 2.4.1 服务端实现

```cpp
// pose_node 提供重载标定的服务
reload_service_ = this->create_service<std_srvs::srv::Trigger>(
    "/pose_node/reload_calibration",
    std::bind(&PoseNode::reloadCalibration, this,
              std::placeholders::_1, std::placeholders::_2)
);

void reloadCalibration(
    const std_srvs::srv::Trigger::Request::SharedPtr /*request*/,
    std_srvs::srv::Trigger::Response::SharedPtr response)
{
    try {
        // 业务逻辑：从 YAML 加载 R/T 矩阵
        pose_solver_->setExtrinsic(R, T);
        is_calibrated_ = true;
        
        response->success = true;
        response->message = "Calibration reloaded successfully";
    } catch (const std::exception& e) {
        response->success = false;
        response->message = std::string("Failed: ") + e.what();
        RCLCPP_ERROR(this->get_logger(), "重载失败: %s", e.what());
    }
}
```

### 2.4.2 客户端实现（嵌套调用）

```cpp
// calibrate_node 中调用 pose_node 的重载服务
bool callPoseNodeReload() {
    auto temp_node = std::make_shared<rclcpp::Node>("_calib_reload_0");
    auto client = temp_node->create_client<std_srvs::srv::Trigger>(
        "/pose_node/reload_calibration");
    
    // 等待服务上线
    if (!client->wait_for_service(std::chrono::seconds(3))) {
        return false;
    }
    
    auto request = std::make_shared<std_srvs::srv::Trigger::Request>();
    auto future = client->async_send_request(request);
    
    // 使用临时 Executor 等待结果
    rclcpp::executors::SingleThreadedExecutor temp_exec;
    temp_exec.add_node(temp_node);
    auto status = temp_exec.spin_until_future_complete(future, std::chrono::seconds(5));
    temp_exec.remove_node(temp_node);
    
    if (status != rclcpp::FutureReturnCode::SUCCESS) {
        return false;
    }
    
    auto result = future.get();
    return result->success;
}
```

**设计要点**：
- 临时节点 + 独立 Executor 避免阻塞主 Executor
- `wait_for_service` 防止服务未上线时死等
- `spin_until_future_complete` 同步等待异步结果

## 2.6 异常处理与健壮性

本项目在所有回调中都包裹了 try-catch，确保单个回调异常不会导致整个节点崩溃：

```cpp
void image_callback(const sensor_msgs::msg::Image::ConstSharedPtr msg) {
    try {
        auto cv_ptr = cv_bridge::toCvShare(msg, "bgr8");
        cv::Mat frame = cv_ptr->image;
        std::vector<Result> results = pipeline_->process(frame);
        // ... 发布结果
    }
    catch (const cv_bridge::Exception& e) {
        RCLCPP_ERROR(this->get_logger(), "cv_bridge 转换失败: %s", e.what());
    }
    catch (const std::exception& e) {
        RCLCPP_ERROR(this->get_logger(), "检测回调异常: %s", e.what());
    }
}
```

## 2.7 进程内零拷贝（Intra-Process Zero-Copy）

### 2.7.1 原理

当发布者和订阅者在**同一进程**中时，ROS2 支持零拷贝：直接传递指向共享内存的指针，避免数据序列化/反序列化和内存拷贝。

### 2.7.2 启用方式

**1. 使用 ComposableNodeContainer（推荐）**

```python
# launch/detect_pipeline.launch.py
pipeline_container = ComposableNodeContainer(
    name='detect_pipeline_container',
    namespace='',
    package='rclcpp_components',
    executable='component_container',  # 单线程容器
    composable_node_descriptions=[
        ComposableNode(
            package='tensorrt_detect',
            plugin='VideoNode',
            name='video_node',
            extra_arguments=[{'use_intra_process_comms': True}],
        ),
        ComposableNode(
            package='tensorrt_detect',
            plugin='DetectNode',
            name='detect_node',
            extra_arguments=[{'use_intra_process_comms': True}],
        ),
        # ... PoseNode, MapNode
    ],
)
```

**2. 发布时使用 `std::make_unique` / `std::move`**

```cpp
auto msg = std::make_unique<sensor_msgs::msg::Image>();
// ... 填充数据
image_pub_->publish(std::move(msg));  // 转移所有权，支持零拷贝
```

**3. 订阅时使用 `ConstSharedPtr`**

```cpp
void image_callback(const sensor_msgs::msg::Image::ConstSharedPtr msg) {
    // msg 是共享指针，底层数据可能被多个订阅者共享
    // 在 intra-process + 单订阅者场景下，可直接使用 msg->data 指针
}
```

### 2.7.3 detect_node 中的浅拷贝优化

```cpp
void image_callback(const sensor_msgs::msg::Image::ConstSharedPtr msg) {
    auto cv_ptr = cv_bridge::toCvShare(msg, "bgr8");
    // 浅拷贝：cv::Mat 与 msg->data 共享同一块内存
    cv::Mat frame = cv_ptr->image;
    
    // 直接在 frame 上画框（会修改底层 msg->data）
    drawDetect(frame, results, cfg_->model.classNames);
    
    // 因为 publish_debug_image_ 是这帧的唯一订阅者，修改是安全的
}
```

⚠️ **注意**：此优化仅在 **intra-process + 单订阅者** 时安全。如果有多个订阅者，应使用 `cv_ptr->image.clone()` 做深拷贝。

## 2.8 多线程 Executor

### 2.8.1 SingleThreadedExecutor（默认）

```cpp
rclcpp::spin(node);  // 单线程顺序执行所有回调
```

适用于：计算密集型、需要避免并发的场景（如本项目核心推理流水线）。

### 2.8.2 MultiThreadedExecutor

```cpp
// calibrate_node 中使用多线程 Executor
rclcpp::executors::MultiThreadedExecutor executor(
    rclcpp::ExecutorOptions(), 2);  // 2 个线程
executor.add_node(node);
executor.spin();
```

适用于：
- Service callback 中需要**同步调用其他 service**（需要另一个线程处理响应）
- I/O 等待型回调（如文件读写、网络请求）

### 2.8.3 临时 Executor

```cpp
// 在回调中创建临时节点和 Executor 执行嵌套任务
rclcpp::executors::SingleThreadedExecutor temp_executor;
temp_executor.add_node(temp_node);
auto status = temp_executor.spin_until_future_complete(
    frame_future, std::chrono::seconds(10));
temp_executor.remove_node(temp_node);
```

## 2.9 本章小结

- 节点继承 `rclcpp::Node`，在构造函数中完成参数声明、对象初始化、通信接口创建
- 发布者使用 `std::make_unique` + `std::move` 支持零拷贝
- 订阅者回调建议包裹 try-catch，增强健壮性
- 服务嵌套调用时，使用临时节点 + 独立 Executor 避免阻塞
- ComposableNode + `use_intra_process_comms` 是本项目的核心性能优化手段
