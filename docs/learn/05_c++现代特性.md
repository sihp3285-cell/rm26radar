## 5.1 智能指针（Smart Pointers）

### 5.1.1 `std::unique_ptr` —— 独占所有权

本项目中最常用的智能指针，用于管理节点内部对象的生命周期。

```cpp
class DetectNode : public rclcpp::Node {
private:
    std::unique_ptr<Config> cfg_;           // 配置管理器
    std::unique_ptr<DetectPipeline> pipeline_;  // 检测流水线
};

// 构造函数中初始化
cfg_ = std::make_unique<Config>(config_dir);
pipeline_ = std::make_unique<DetectPipeline>(*cfg_);
```

**特点**：
- 独占所指向的对象，不能拷贝，只能 `std::move` 转移
- 离开作用域时自动析构，防止内存泄漏
- `std::make_unique`（C++14）比 `new` 更安全，异常安全

### 5.1.2 `std::shared_ptr` —— 共享所有权

ROS2 的消息系统大量使用 `shared_ptr`：

```cpp
// 订阅回调接收的是 ConstSharedPtr
void image_callback(const sensor_msgs::msg::Image::ConstSharedPtr msg);

// 发布时转移所有权
auto msg = std::make_unique<sensor_msgs::msg::Image>();
image_pub_->publish(std::move(msg));
```

**`ConstSharedPtr` 的妙处**：
- 支持进程内零拷贝：多个订阅者共享同一块底层数据
- `const` 保证订阅者不会意外修改数据
- 引用计数自动管理内存释放

## 5.2 Lambda 表达式

Lambda 在本项目中被广泛用于回调绑定和局部逻辑封装。

### 5.2.1 基本语法

```cpp
[捕获列表](参数列表) -> 返回类型 { 函数体 }
```

### 5.2.2 作为 ROS2 回调

```cpp
// video_node 中的暂停服务
pause_service_ = this->create_service<std_srvs::srv::SetBool>(
    "/video_node/set_pause",
    [this](const std_srvs::srv::SetBool::Request::SharedPtr request,
           std_srvs::srv::SetBool::Response::SharedPtr response) {
        is_paused_ = request->data;
        response->success = true;
        response->message = request->data ? "视频已暂停" : "视频已恢复";
    });
```

**捕获列表 `[this]`**：捕获当前对象的指针，允许在 Lambda 中访问成员变量。

### 5.2.3 作为 Qt 信号槽

```cpp
// qt_display_node.cpp
connect(calibrate_button_, &QPushButton::clicked, this, [this]() {
    if (calibrate_cb_) {
        setCalibrateButtonsEnabled(false);
        calibrate_cb_();
    }
});
```

### 5.2.4 存储为 std::function

```cpp
class DisplayWindow : public QMainWindow {
    std::function<void()> close_cb_;
    std::function<void(bool)> team_flip_cb_;
    std::function<void()> calibrate_cb_;

```

## 5.3 多线程与同步

### 5.3.1 `std::thread`

```cpp
class DetectPipeline {
private:
    std::thread airplaneThread_;  // 异步无人机检测线程
};

// 启动线程
airplaneThread_ = std::thread(&DetectPipeline::airplaneThreadLoop, this);

// 析构时 joining
DetectPipeline::~DetectPipeline() {
    stopThread_ = true;
    airplaneCv_.notify_all();
    if (airplaneThread_.joinable()) {
        airplaneThread_.join();
    }
}
```

### 5.3.2 `std::mutex` 与 `std::lock_guard`

```cpp
public:
    bool Detect(const cv::Mat &frame) {
        std::lock_guard<std::mutex> lock(mtx_);

```

### 5.3.3 `std::condition_variable`

```cpp
// 生产者-消费者模型：DetectPipeline 主线程生产图像，airplaneThread 消费
std::mutex frameMutex_;
std::condition_variable airplaneCv_;
cv::Mat latestFrame_;
bool newFrameAvailable_ = false;

// 生产者（主线程）
{
    std::lock_guard<std::mutex> lock(frameMutex_);
    latestFrame_ = frame.clone();
    newFrameAvailable_ = true;
}
airplaneCv_.notify_one();

// 消费者（airplaneThread）
void airplaneThreadLoop() {
    while (!stopThread_) {
        std::unique_lock<std::mutex> lock(frameMutex_);
        airplaneCv_.wait(lock, [this] { return newFrameAvailable_ || stopThread_; });
        if (stopThread_) break;
        cv::Mat frame = latestFrame_;
        newFrameAvailable_ = false;
        lock.unlock();
        
        // 执行无人机检测...
    }
}
```

### 5.3.4 `std::atomic`

```cpp
class DetectPipeline {
    std::atomic<bool> stopThread_{false};     // 线程停止标志
    std::atomic<double> lastAirplaneMs_{0.0}; // 耗时统计（无锁读写）
};

class VideoNode {
    std::atomic<bool> is_paused_{false};      // 暂停标志（多线程安全）
};
```

**`atomic` vs `mutex`**：
- `atomic`：无锁，适用于简单类型（bool、int、float）的读写
- `mutex`：有锁，适用于保护复杂数据结构或代码块

## 5.4 `std::promise` 与 `std::future`

用于在 ROS2 中同步获取异步结果：

```cpp
// calibrate_node 中：临时节点订阅图像，需要同步等待第一帧
std::promise<cv::Mat> frame_promise;
auto frame_future = frame_promise.get_future();

auto image_sub = temp_node->create_subscription<sensor_msgs::msg::Image>(
    image_topic_, rclcpp::QoS(1),
    [&](const sensor_msgs::msg::Image::SharedPtr msg) {
        try {
            auto cv_ptr = cv_bridge::toCvCopy(msg, "bgr8");
            frame_promise.set_value(cv_ptr->image.clone());
        } catch (...) {
            frame_promise.set_exception(std::current_exception());
        }
    });

// 同步等待（最多 10 秒）
auto status = temp_executor.spin_until_future_complete(
    frame_future, std::chrono::seconds(10));

if (status == rclcpp::FutureReturnCode::SUCCESS) {
    cv::Mat captured_frame = frame_future.get();
}
```

## 5.5 C++17 特性

### 5.5.1 结构化绑定（Structured Bindings）

```cpp
// C++17 之前
std::pair<bool, std::string> result = doCalibration();
bool success = result.first;
std::string msg = result.second;

// C++17：结构化绑定
auto [success, msg] = doCalibration();

// 也可用于返回多个值
auto [r, g, b] = getPixelColor(x, y);
```

### 5.5.2 `if` 语句初始化器

```cpp
// C++17 之前
std::lock_guard<std::mutex> lock(mtx_);
if (frame_ready) { ... }

// C++17：锁的作用域限制在 if 块内
if (std::lock_guard<std::mutex> lock(mtx_); frame_ready) {
    // lock 在这里自动释放
}
```

### 5.5.3 `std::filesystem`

```cpp
#include <filesystem>

std::filesystem::path dir(config_dir_);
std::string camera_yaml = (dir / "camera.yaml").string();
std::string calibPath = (dir / "calib_result.yaml").string();

if (std::filesystem::exists(calibPath)) {
    // ...
}
```

**替代了 C++17 之前需要 Boost.Filesystem 或 POSIX API 的繁琐操作。**

## 5.8 本章小结

| 特性 | 本项目典型应用 |
|:---|:---|
| `std::unique_ptr` | 节点内部业务对象的生命周期管理 |
| `std::shared_ptr` / `ConstSharedPtr` | ROS2 消息零拷贝传递 |
| Lambda | ROS2 回调、Qt 信号槽、局部逻辑封装 |
| `std::thread` + `condition_variable` | 异步无人机检测线程 |
| `std::atomic` | 无锁的暂停标志、耗时统计 |
| `std::promise/future` | Service 嵌套调用中的同步等待 |
| 结构化绑定 | `doCalibration()` 返回多值解析 |
| `std::filesystem` | 配置文件路径拼接与存在性检查 |
| `constexpr` | Tracker 固定槽位数量等编译期常量 |
