# 10 Qt5 可视化开发

> 本项目使用 Qt5 构建了一个完整的交互式显示界面 `qt_display_node`，集成视频流显示、小地图、状态栏、操作按钮等功能。Qt 与 ROS2 的融合需要解决**事件循环冲突**和**线程安全**两大核心问题。本章深入讲解实现细节。

## 10.1 Qt5 基础

### 10.1.1 核心概念

| 概念 | 说明 | 本项目应用 |
|:---|:---|:---|
| `QApplication` | 全局应用对象，管理事件循环 | `main()` 中创建 |

## 10.2 ROS2 + Qt 的线程模型

### 10.2.1 问题：双事件循环

- **Qt**：`QApplication::exec()` 运行 Qt 事件循环（主线程）
- **ROS2**：`rclcpp::spin()` 运行 ROS2 事件循环（需要独立线程）

如果两者在同一个线程运行，会互相阻塞。

### 10.2.2 解决方案：分离线程

```cpp
int main(int argc, char *argv[]) {
    rclcpp::init(argc, argv);
    QApplication app(argc, argv);
    
    DisplayWindow window;
    window.show();
    
    auto node = std::make_shared<QtDisplayNode>(&window);
    
    // Qt 定时器在主线程驱动 UI 刷新
    QTimer timer;
    QObject::connect(&timer, &QTimer::timeout, &window, &DisplayWindow::refresh);
    timer.start(33);  // 33 ms ≈ 30 FPS
    
    // ROS2 spin 在独立线程
    std::thread ros_thread([node]() {
        rclcpp::spin(node);
        QApplication::quit();  // ROS 退出时结束 Qt
    });
    
    int ret = app.exec();  // 主线程运行 Qt 事件循环
    if (rclcpp::ok()) rclcpp::shutdown();
    ros_thread.join();
    return ret;
}
```

**线程分工**：

| 线程 | 职责 |
|:---|:---|
| 主线程（Qt） | UI 渲染、用户交互、定时刷新 |
| ROS 线程 | 订阅消息、Service 处理、发布消息 |

### 10.2.3 线程安全：QMutex 保护共享数据

```cpp
class QtDisplayNode : public rclcpp::Node {
private:
    QMutex mutex_;
    cv::Mat latest_frame_;      // ROS 线程写入，Qt 线程读取
    cv::Mat latest_map_;
    
public:
    // ROS 回调（ROS 线程）
    void video_callback(const sensor_msgs::msg::Image::SharedPtr msg) {
        auto cv_ptr = cv_bridge::toCvCopy(msg, "bgr8");
        QMutexLocker lock(&mutex_);
        latest_frame_ = cv_ptr->image.clone();  // 深拷贝 + 加锁
    }
    
    // Qt 读取（主线程）
    void fetchData(cv::Mat &frame, cv::Mat &map, ...) {
        QMutexLocker lock(&mutex_);
        frame = latest_frame_.clone();
        map = latest_map_.clone();
    }
};
```

### 10.2.4 跨线程 UI 更新：QMetaObject::invokeMethod

ROS 线程不能直接操作 Qt 控件（非线程安全），必须通过 `invokeMethod` 将操作投递到 Qt 主线程：

```cpp
// ROS 线程中调用 Service 后更新 UI
QMetaObject::invokeMethod(window_, [this, operation_name, success, msg]() {
    QString status_text = success
        ? QString::fromStdString(operation_name + " 成功: " + msg)
        : QString::fromStdString(operation_name + " 失败: " + msg);
    window_->showOperationStatus(status_text, success);
    window_->setCalibrateButtonsEnabled(true);
});
```

### 10.3.3 OpenCV Mat → QPixmap

```cpp
static QPixmap cvMatToQPixmap(const cv::Mat &cv_img) {
    cv::Mat rgb;
    if (cv_img.channels() == 3)
        cv::cvtColor(cv_img, rgb, cv::COLOR_BGR2RGB);
    else if (cv_img.channels() == 4)
        cv::cvtColor(cv_img, rgb, cv::COLOR_BGRA2RGB);
    else
        rgb = cv_img;
    
    // QImage 不拥有内存，必须用 .copy() 创建副本
    QImage img(rgb.data, rgb.cols, rgb.rows, 
               static_cast<int>(rgb.step), QImage::Format_RGB888);
    return QPixmap::fromImage(img.copy());
}
```

                      const std::string &operation_name) {
    if (is_operation_running_.exchange(true)) {
        return;  // 防止重复触发

## 10.5 本章小结

| 技术点 | 解决方案 | 关键代码 |
|:---|:---|:---|
| 双事件循环 | ROS spin 放独立线程 | `std::thread( rclcpp::spin )` |
| 线程安全共享数据 | `QMutex` + `QMutexLocker` | `fetchData()` / 回调中加锁 |
| 跨线程 UI 更新 | `QMetaObject::invokeMethod` | Service 结果回调中调用 |
| 定时刷新 | `QTimer` | `timer.start(33)` |
| Mat → Qt 图像 | `QImage` + `QPixmap::fromImage` | `cvMatToQPixmap()` |
| 异步 Service | 后台 `std::thread` + `detach` | `callServiceAsync()` |
| 样式定制 | QSS（Qt StyleSheet） | `setStyleSheet()` |
