# 03 ROS2 消息与接口

> ROS2 的消息系统基于接口定义文件（`.msg`、`.srv`、`.action`），通过 `rosidl` 代码生成器自动生成 C++ 和 Python 的绑定类。本章详解本项目的自定义消息设计、`cv_bridge` 图像转换，以及消息序列化与零拷贝的底层机制。

---

## 3.1 自定义消息包结构

本项目的自定义消息定义在 `tensorrt_detect_msgs` 包中：

```
tensorrt_detect_msgs/
├── msg/
│   ├── DetectionBox.msg      # 单个检测框
│   ├── DetectionArray.msg    # 检测框数组
│   ├── WorldTarget.msg       # 世界坐标目标
│   ├── WorldTargetArray.msg  # 世界坐标目标数组
│   ├── RadarMap.msg          # 雷达地图坐标
│   ├── MapTactics.msg        # 地图战术分析
│   └── PipelineTiming.msg    # 流水线耗时统计
├── package.xml
└── CMakeLists.txt
```

## 3.2 `.msg` 文件语法

### 3.2.1 基础类型

```msg
# DetectionBox.msg
int32 idx              # 类别索引
float32 confidence     # 置信度
int32 x                # 左上角 x
int32 y                # 左上角 y
int32 width
int32 height
int32 armor_color      # 装甲板颜色（队伍）
bool is_dead           # 是否被击毁
float32 world_x        # 世界坐标 X
float32 world_y        # 世界坐标 Z（这里命名为 y）
float32 fps            # 当前帧率
```

### 3.2.2 数组与嵌套

```msg
# DetectionArray.msg
std_msgs/Header header      # 标准消息头（时间戳 + 坐标系）
DetectionBox[] detections   # 动态数组
```

```msg
# RadarMap.msg
std_msgs/Header header
float32[6] blue_x           # 固定大小数组：蓝方 6 个单位
float32[6] blue_y
float32[6] red_x
float32[6] red_y
```

### 3.2.3 标准消息头（Header）

`std_msgs/Header` 是 ROS2 的通用消息头：

```msg
# std_msgs/Header
builtin_interfaces/Time stamp    # 时间戳
string frame_id                  # 坐标系名称
```

**本项目中的使用惯例**：
- `stamp`：复用上游图像的时间戳，便于计算端到端延迟
- `frame_id`：标记消息来源，如 `"detection"`、`"radar_map"`

## 3.3 消息包的 CMake 配置

```cmake
# tensorrt_detect_msgs/CMakeLists.txt
cmake_minimum_required(VERSION 3.22.1)
project(tensorrt_detect_msgs)

find_package(ament_cmake REQUIRED)
find_package(rosidl_default_generators REQUIRED)
find_package(std_msgs REQUIRED)

# 声明接口文件
rosidl_generate_interfaces(${PROJECT_NAME}
  msg/DetectionBox.msg
  msg/DetectionArray.msg
  msg/WorldTarget.msg
  msg/WorldTargetArray.msg
  msg/RadarMap.msg
  msg/MapTactics.msg
  msg/PipelineTiming.msg
  DEPENDENCIES std_msgs   # 依赖的其他消息包
)

ament_export_dependencies(rosidl_default_runtime)
ament_package()
```

编译后，ROS2 会自动生成以下 C++ 类：
- `tensorrt_detect_msgs::msg::DetectionBox`
- `tensorrt_detect_msgs::msg::DetectionArray`
- ...

## 3.4 消息的使用

### 3.4.1 创建与填充

```cpp
#include "tensorrt_detect_msgs/msg/detection_array.hpp"
#include "tensorrt_detect_msgs/msg/detection_box.hpp"

auto armor_msg = std::make_unique<tensorrt_detect_msgs::msg::DetectionArray>();
armor_msg->header = msg->header;           // 复用时间戳
armor_msg->header.frame_id = "detection";
armor_msg->detections.reserve(results.size());

for (const auto& res : results) {
    tensorrt_detect_msgs::msg::DetectionBox box;
    box.idx = res.idx;
    box.confidence = res.confidence;
    box.x = res.box.x;
    box.y = res.box.y;
    box.width = res.box.width;
    box.height = res.box.height;
    box.armor_color = res.armorColor;
    box.is_dead = res.isDead;
    box.world_x = res.worldPoint.x;
    box.world_y = res.worldPoint.y;
    armor_msg->detections.push_back(box);
}

armor_pub_->publish(std::move(armor_msg));
```

## 3.5 cv_bridge：ROS 图像消息与 OpenCV 的桥梁

### 3.5.1 基本概念

`cv_bridge` 是 ROS2 与 OpenCV 之间的转换库，处理两种图像表示：
- **ROS**: `sensor_msgs::msg::Image` —— 元数据 + 原始字节数组
- **OpenCV**: `cv::Mat` —— 带引用计数的矩阵头 + 数据指针

### 3.5.2 toCvShare（零拷贝，推荐）

```cpp
#include <cv_bridge/cv_bridge.hpp>

void image_callback(const sensor_msgs::msg::Image::ConstSharedPtr msg) {
    // toCvShare: 返回的 cv::Mat 与 msg->data 共享内存
    auto cv_ptr = cv_bridge::toCvShare(msg, "bgr8");
    cv::Mat frame = cv_ptr->image;  // 浅拷贝
    
    // 在 frame 上的修改会直接反映到 msg->data
    // 仅适用于 intra-process + 单订阅者场景
}
```

### 3.5.3 toCvCopy（深拷贝，安全）

```cpp
// qt_display_node 中需要把图像保存到节点成员变量，必须使用深拷贝
auto cv_ptr = cv_bridge::toCvCopy(msg, "bgr8");
QMutexLocker lock(&mutex_);
latest_frame_ = cv_ptr->image.clone();  // 深拷贝
```

### 3.5.4 CvImage → ImageMsg

```cpp
// OpenCV Mat 转 ROS Image 消息
std_msgs::msg::Header header;
header.stamp = this->now();
header.frame_id = "detected_frame";

auto out_msg = std::make_unique<sensor_msgs::msg::Image>();
cv_bridge::CvImage(header, "bgr8", debug_output_frame_).toImageMsg(*out_msg);
image_pub_->publish(std::move(out_msg));
```

### 3.5.5 编码格式对照

| OpenCV | ROS encoding | 说明 |
|:---|:---|:---|
| `CV_8UC3` (BGR) | `"bgr8"` | 最常用 |
| `CV_8UC3` (RGB) | `"rgb8"` | — |
| `CV_8UC4` (BGRA) | `"bgra8"` | 带透明通道 |
| `CV_8UC1` | `"mono8"` | 灰度图 |
| `CV_16UC1` | `"16UC1"` | 深度图 |

## 3.6 消息设计的最佳实践

### 3.6.1 时间戳传递

本项目采用**时间戳透传**策略，便于端到端延迟分析：

```
video_node 采集图像
    ↓ stamp = now()
detect_node 处理
    ↓ header 原样复制
pose_node 解算
    ↓ header 原样复制
map_node 绘制
```

### 3.6.2 固定数组 vs 动态数组

```msg
# 固定数组：适合已知数量的实体（如双方各 6 个机器人）
float32[6] blue_x

# 动态数组：适合数量不定的检测结果
DetectionBox[] detections
```

### 3.6.3 使用 frame_id 标记来源

```cpp
// detect_node
armor_msg->header.frame_id = "detection";

// map_node
header.frame_id = "radar_map";
```

这在多传感器融合、调试 RViz 显示时非常重要。

## 3.7 本章小结

- `.msg` 文件定义数据结构，ROS2 自动生成 C++ 类
- `std_msgs/Header` 用于时间戳和坐标系标记
- `cv_bridge::toCvShare` 零拷贝但需注意线程安全；`toCvCopy` 安全但开销大
- 自定义消息包需要 `rosidl_generate_interfaces` 和 `ament_export_dependencies`
- 消息设计要考虑：时间戳透传、固定/动态数组选择、frame_id 语义
