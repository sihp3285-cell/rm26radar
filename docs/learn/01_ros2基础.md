# 01 ROS2 基础

> ROS2（Robot Operating System 2）是机器人软件开发的主流中间件框架。本项目使用 **ROS2 Humble**（或更新版本），所有节点均基于 `rclcpp`（ROS Client Library for C++）构建。

---

## 1.1 ROS2 与 ROS1 的核心区别

| 特性 | ROS1 | ROS2 |
|:---|:---|:---|
| 中间件 | 自定义 TCPROS/UDPROS | **DDS**（Data Distribution Service） |
| 节点启动 | 必须启动 roscore | **无中心节点**，分布式发现 |
| 构建系统 | catkin | **ament_cmake / colcon** |
| C++ 标准 | C++11 | **C++14/17** |
| 实时性 | 弱 | 支持实时嵌入式 |
| 多机通信 | 需配置 ROS_MASTER_URI | **自动发现**，支持局域网多机 |

在本项目中，DDS 的可靠性QoS、进程内零拷贝（intra-process zero-copy）被大量使用以提升图像传输效率。

## 1.2 核心概念

### 1.2.1 节点（Node）

节点是 ROS2 中最小的计算单元，一个进程可包含多个节点。

```cpp
// 本项目中所有节点的基类都是 rclcpp::Node
class DetectNode : public rclcpp::Node {
public:
    explicit DetectNode(const rclcpp::NodeOptions& options = rclcpp::NodeOptions())
        : Node("detect_node", options)  // 节点名称
    {
        RCLCPP_INFO(this->get_logger(), "DetectNode 初始化完成");
    }
};
```

### 1.2.2 话题（Topic）—— 发布/订阅模型

话题是节点间异步通信的通道，采用**多对多**的发布/订阅模型。

```cpp
// 发布者（Publisher）
image_pub_ = this->create_publisher<sensor_msgs::msg::Image>(
    "/detected_image",           // 话题名
    rclcpp::QoS(1)               // QoS 策略
);

// 订阅者（Subscription）
image_sub_ = this->create_subscription<sensor_msgs::msg::Image>(
    "/image_raw",
    rclcpp::QoS(1),
    std::bind(&DetectNode::image_callback, this, std::placeholders::_1)
);
```

本项目中的话题拓扑：

```
/video_node/image_raw
    ↓
/detect_node/detected_image
/detect_node/armor_detections
    ↓
/pose_node/world_targets
    ↓
/map_node/map_image
/map_node/radar_map
/map_node/map_tactics
```

### 1.2.3 服务（Service）—— 请求/响应模型

服务适用于需要**即时反馈**的同步调用场景，如标定触发、配置重载。

```cpp
// 服务端（Service Server）
reload_service_ = this->create_service<std_srvs::srv::Trigger>(
    "/pose_node/reload_calibration",
    std::bind(&PoseNode::reloadCalibration, this,
              std::placeholders::_1, std::placeholders::_2)
);

// 客户端（Service Client）—— 在 calibrate_node 中调用 pose_node 的重载服务
auto client = temp_node->create_client<std_srvs::srv::Trigger>(
    "/pose_node/reload_calibration"
);
client->wait_for_service(std::chrono::seconds(3));
auto future = client->async_send_request(request);
```

### 1.2.4 参数（Parameter）

ROS2 参数是节点的运行时配置，可在 Launch 文件中覆盖。

```cpp
// 声明参数（带默认值）
this->declare_parameter<std::string>("config_dir", "/home/delphine/rm/tensorrt10_detect/configs");
this->declare_parameter<std::string>("input_topic", "/image_raw");
this->declare_parameter<bool>("publish_debug_image", true);

// 读取参数
std::string config_dir = this->get_parameter("config_dir").as_string();
bool publish_debug = this->get_parameter("publish_debug_image").as_bool();
```

对应 `launch/detect_pipeline.launch.py` 中的参数传递：

```python
params_file = PathJoinSubstitution([
    FindPackageShare('tensorrt_detect'), 'config', 'ros2_params.yaml',
])

ComposableNode(
    package='tensorrt_detect',
    plugin='DetectNode',
    name='detect_node',
    parameters=[params_file],   # ← 参数文件注入
    extra_arguments=[{'use_intra_process_comms': True}],
)
```

## 1.3 QoS（Quality of Service）

ROS2 基于 DDS 提供了丰富的 QoS 策略，用于控制通信的可靠性、实时性和资源占用。

### 1.3.1 常用 QoS 配置

```cpp
// 1. 传感器数据：best_effort + 浅队列，允许丢帧保实时
rclcpp::QoS(1).best_effort();

// 2. 可靠传输：reliable，不允许丢数据
rclcpp::QoS(10).reliable();

// 3. 保持历史（Keep Last N）
rclcpp::QoS(10).keep_last(10);
```

### 1.3.2 本项目中的 QoS 实践

```cpp
// video_node 发布图像：队列深度 1，最新帧最重要
image_pub_ = this->create_publisher<sensor_msgs::msg::Image>(
    topic_name_, rclcpp::QoS(1));

// detect_node 发布检测数组：best_effort，高吞吐
armor_pub_ = this->create_publisher<tensorrt_detect_msgs::msg::DetectionArray>(
    "/armor_detections", rclcpp::QoS(10).best_effort());

// qt_display_node 订阅战术消息：reliable，不能丢
team_flip_pub_ = this->create_publisher<std_msgs::msg::Bool>(
    "/flip_team", rclcpp::QoS(1).reliable());
```

| 场景 | QoS 选择 | 理由 |
|:---|:---|:---|
| 视频流 | `QoS(1)` / `best_effort` | 实时性优先，旧帧无意义 |
| 检测结果 | `QoS(10).best_effort()` | 高频数据，允许偶尔丢包 |
| 标定/重载服务 | `reliable()` | 命令必须送达 |
| 阵营切换 | `QoS(1).reliable()` | 状态切换不能丢失 |

## 1.5 ROS2 日志系统

ROS2 提供分级日志，支持节流输出，避免刷屏。

```cpp
// 普通信息
RCLCPP_INFO(this->get_logger(), "DetectNode 初始化完成");

// 警告（带节流：每 10 秒最多输出一次）
RCLCPP_WARN_THROTTLE(
    this->get_logger(),
    *this->get_clock(),
    10000,  // 毫秒
    "标定未就绪，跳过世界坐标计算"
);

// 错误
RCLCPP_ERROR(this->get_logger(), "cv_bridge 转换失败: %s", e.what());

// 调试（仅在 Debug 构建输出）
RCLCPP_DEBUG(this->get_logger(), "详细调试信息");
```

| 宏 | 级别 | 用途 |
|:---|:---|:---|
| `RCLCPP_DEBUG` | DEBUG | 详细调试 |
| `RCLCPP_INFO` | INFO | 常规状态 |
| `RCLCPP_WARN` | WARN | 非致命异常 |
| `RCLCPP_ERROR` | ERROR | 错误，功能受影响 |
| `RCLCPP_FATAL` | FATAL | 致命错误，即将崩溃 |

**节流宏**（`RCLCPP_*_THROTTLE`）在处理高频回调（如每帧图像）时特别有用，避免日志淹没控制台。

## 1.6 常用 ROS2 CLI 工具

```bash
# 查看节点列表
ros2 node list

# 查看话题列表
ros2 topic list

# 查看话题带宽和频率
ros2 topic hz /image_raw
ros2 topic bw /armor_detections

# 查看话题类型
ros2 topic info /armor_detections

# 手动发布测试消息
ros2 topic pub /flip_team std_msgs/msg/Bool "data: true"

# 调用服务
ros2 service call /calibration/start std_srvs/srv/Trigger

# 查看参数
ros2 param list /detect_node
ros2 param get /detect_node publish_debug_image

# 录制/回放 bag
ros2 bag record /image_raw /armor_detections
ros2 bag play <bag_file>
```

## 1.7 本章小结

- ROS2 基于 DDS，无中心节点，支持分布式和多机通信
- 通信方式：**Topic**（异步 Pub/Sub）、**Service**（同步 Req/Res）、**Parameter**（配置）
- QoS 是 ROS2 的核心优势，需根据场景（实时 vs 可靠）灵活选择
- `ament_cmake` + `colcon` 是 ROS2 的构建工具链
- 善用 `RCLCPP_*_THROTTLE` 控制高频日志输出
