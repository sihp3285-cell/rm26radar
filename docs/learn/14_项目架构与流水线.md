## 14.2 节点职责详解

| 节点 | 进程 | 输入话题 | 输出话题 | 核心职责 |
|:---|:---|:---|:---|:---|
| **VideoNode** | Container | 参数 `video_path` | `/image_raw` | 读取视频/摄像头，按固定帧率发布图像 |
| **DetectNode** | Container | `/image_raw` | `/detected_image`, `/armor_detections`, `/pipeline_timing` | TensorRT 多模型级联推理 |
| **PoseNode** | Container | `/armor_detections` | `/world_targets` | PnP 位姿解算 + Tracker 多目标跟踪 |
| **MapNode** | Container | `/world_targets` | `/map_image`, `/radar_map`, `/map_tactics` | 小地图绘制 + 战术分析 |
| **DisplayNode** | 独立 | `/detected_image` | 无 | OpenCV 窗口显示（调试） |
| **CalibrateNode** | 独立 | `/image_raw` (临时订阅) | `/calibration/start` (Service) | 交互式相机标定 |
| **ROISetNode** | 独立 | `/image_raw` (临时订阅) | `/roi_set/start` (Service) | 交互式 ROI 框定 |
| **QtDisplayNode** | 独立 | `/detected_image`, `/map_image`, `/armor_detections`, `/map_tactics`, `/pipeline_timing` | `/flip_team` | Qt GUI 主界面 |

## 14.3 DetectPipeline 级联推理

`DetectPipeline` 是系统的核心算法类，实现了多模型级联检测：

```
输入帧
    ↓
[robot_only.engine] ──> 车辆检测（1280×1280）
    ↓
对每个检测到的车辆 ROI
    ↓
[newarmor.engine] ──> 装甲板检测（192×192）
    ↓
对每个检测到的装甲板 ROI
    ↓
[classify_hku.engine] ──> 装甲板分类（64×64）
    ↓
输出：带类别和置信度的检测框

─────────────────────────────────────────────
异步分支（独立线程）：
[airplane640.engine] ──> 无人机检测（640×640，每 200ms）
```

### 14.3.1 代码结构

```cpp
class DetectPipeline {
public:
    std::vector<Result> process(const cv::Mat& frame);
    
private:
    Model detectModel_;      // 车辆检测
    Model armorDetector_;    // 装甲板检测
    Model classifyModel_;    // 装甲板分类
    std::unique_ptr<Model> airplaneModel_;  // 无人机检测（可选）
    
    std::vector<Result> runDetect(const cv::Mat& frame);
    std::vector<Result> runArmorDetect(const cv::Mat& frame,
                                       const std::vector<Result>& detections);
    std::vector<Result> runArmorDetectBatch(const cv::Mat& frame,
                                             const std::vector<Result>& detections,
                                             std::vector<Result>* outposts = nullptr);
    void runClassifyBatch(const cv::Mat& frame, std::vector<Result>& detections);
    std::vector<Result> runOutpostDetect(const cv::Mat& frame);
    std::vector<Result> runAirplaneDetect(const cv::Mat& frame);
    
    // 异步线程
    void airplaneThreadLoop();
    std::thread airplaneThread_;
};
```

### 14.3.2 Batch 推理优化

本项目对装甲板分类使用了 **Batch 推理**：

```cpp
// 传统：逐个推理（N 次 API 调用，N 次 memcpy）
for (const auto& roi : rois) {
    int cls = classifyModel_.predictClass(roi);  // 慢
}

// 优化：Batch 推理（1 次 API 调用，1 次 memcpy）
std::vector<int> classes = classifyModel_.predictClassBatch(rois);  // 快
```

**性能提升**：Batch 推理减少了 CUDA kernel 启动开销和内存传输次数，通常有 **2~5 倍**加速。

## 14.4 数据流时间线

```
t=0ms   VideoNode 读取帧，stamp=now()
t=1ms   → /image_raw 发布（零拷贝）
t=2ms   DetectNode 收到帧
        ├── car_ms: 3ms   (车辆检测)
        ├── armor_ms: 2ms (装甲板检测)
        ├── cls_ms: 1ms   (分类，Batch)
        └── total_ms: 6ms
t=8ms   → /armor_detections 发布（复用 stamp）
t=9ms   PoseNode 收到检测
        ├── 解算世界坐标: 1ms
        ├── Tracker 更新: 0.5ms
        └── total: 1.5ms
t=11ms  → /world_targets 发布
t=12ms  MapNode 收到目标
        ├── 地图绘制: 2ms
        └── 战术分析: 0.5ms
t=15ms  → /map_image 发布
t=16ms  QtDisplayNode 刷新 UI

端到端延迟 ≈ 16ms（约 1 帧 @ 60 FPS）
```

## 14.5 战术分析（MapAnalyzer）

```cpp
class MapAnalyzer {
public:
    void evaluate(const std::vector<tensorrt_detect_msgs::msg::WorldTarget>& targets);
    
    int engineer_on_island() const;      // 0=无, 1=我方, 2=敌方
    bool opponent_attack() const;        // 敌方大攻
    bool our_attack() const;             // 我方大攻
    int opponent_near_fortress() const;  // 敌方接近堡垒
};
```

战术分析基于世界坐标进行几何判断：
- **工程上岛**：工程机器人在特定区域
- **大攻**：多个敌方机器人越过中场
- **接近堡垒**：敌方机器人进入堡垒警戒圈

## 14.6 异常处理与降级策略

| 异常场景 | 处理策略 |
|:---|:---|
| 标定文件缺失 | PoseNode 跳过解算，持续提示等待标定 |
| mesh 加载失败 | PoseNode 使用平面 fallback（world_y = 0） |
| 视频源断开 | VideoNode 循环播放（`cap.set(CAP_PROP_POS_FRAMES, 0)`） |
| TensorRT 推理失败 | 打印错误，返回空结果，不崩溃 |
| 检测结果为空 | Tracker 进入预测状态，维持历史位置 |
| Service 调用超时 | 返回失败响应，UI 提示用户重试 |

## 14.8 本章小结

| 层面 | 设计要点 |
|:---|:---|
| **架构** | 8 个节点分工明确，4 个在零拷贝容器，4 个独立进程 |
| **推理** | 4 个 TensorRT 模型级联 + Batch 优化 |
| **跟踪** | 固定槽位 + 双 Kalman + BotIdentity 稳定 |
| **标定** | 交互式 PnP，支持运行时重载 |
| **可视化** | OpenCV 调试 + Qt5 主界面 |
| **战术** | 基于世界坐标的几何规则判断 |
| **异常** | 多层降级，单点故障不影响整体 |
